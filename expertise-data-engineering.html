Data Engineering & Cloud

Building reliable data foundations for analytics & machine learning

I focus on designing and operating cloud-based data pipelines that make downstream analytics and machine learning possible. My experience ranges from ingesting raw sensor and enterprise data to modeling clean layers for BI dashboards and ML training on Azure.

What I’m strong at in Data Engineering

ETL/ELT Pipeline Design

Experience designing batch and near real-time pipelines using Azure Data Factory, Databricks, and Synapse. I care about reliability, idempotency, and clear separation between raw, cleaned, and curated layers.

Azure-Centric Architectures

Hands-on with ADF, Synapse, Azure Databricks, Azure Storage, and Azure ML integration. I’m comfortable building end-to-end solutions on the Azure stack for enterprise workloads.

Data Modeling & Performance

Designing staging, normalized, and reporting layers; using star-schema like structures for BI; and focusing on indexing, partitioning, and efficient query design in SQL and Spark.

Operational Mindset

Monitoring pipeline health, handling failures gracefully, and working with logging/alerts. I aim for pipelines that teams can trust on a daily basis.

Standard Data Engineering Workflow I Use

Across roles and projects, I’ve converged on a repeatable process for designing and implementing data pipelines that support both BI and ML workloads.

1

Understand Sources & Consumers

Map data sources (sensors, APIs, databases, files) and consumers (dashboards, reports, ML models). Clarify freshness requirements, SLAs, security constraints, and data ownership.

Requirements & data discovery
2

Ingestion & Landing Zones

Set up ingestion pipelines using ADF and Databricks to bring data into a raw/landing zone (e.g., Azure Data Lake). Ensure schema, partitioning, and metadata capture are well-defined.

Raw data layer
3

Cleaning, Standardization & Enrichment

Use PySpark/SQL transforms to clean, normalize, and enrich data. Handle missing values, time alignment, and unit conversions; create business-friendly fields and dimensions.

Cleaned/bronze-silver layers
4

Curated & Analytics-Ready Models

Build curated tables or views optimized for analytics and ML: star-schemas, wide tables for reporting, and feature tables for ML training in Databricks or Azure ML.

Curated/gold layer
5

Orchestration, Scheduling & Monitoring

Use ADF pipelines and Databricks jobs to orchestrate the workflow, define dependencies, and configure alerts/logging. Make sure failures are visible and recoverable.

Production readiness
6

Handover & Documentation

Document the architecture, data contracts, and operational runbooks. Collaborate with BI, ML, and platform teams so that the platform is maintainable and extensible.

Sustainable operations
